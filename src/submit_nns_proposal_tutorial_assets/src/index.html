"How to Submit NNS Proposals" for Dummies

Create, sign, and submit Motion proposals to the NNS.

The Network Nervous System (NNS) is the controlling mechanism by which the Internet Computerâ€™s nodes and subnets are organized, tracked, and managed. The Internet Computer community is actively involved in designing and shaping the network's roadmap. Proposed upgrades to the NNS are subject to community discussion, voting, and approval. The community of ICP token holders can use neurons to vote on proposed roadmap items. Check out the Internet Computer Roadmap page here. This walkthrough will detail how to submit a motion proposal.

Install Keysmith

Keysmith is a tool that lets you derive cryptographic keys and identifiers for the Internet Computer. Among these identifiers includes an account identifier, which indicates the source or destination of an ICP token transfer. Keysmith does not sign or send messages to the Internet Computer.

To install Keysmith...

1.1 Navigate to the Keysmith repo on Github: https://github.com/dfinity/keysmith

Tip: To install on a Macbook, choose keysmith-darwin-amd64.tar.gz from the options.

1.2 Follow the download instructions

1.3 Follow the verify instructions

1.4 Follow the install instructions

Terminal
Terminal
keysmith version
1.4.1-release

You can confirm successful installation by checking the version number (Note: numbers will vary as new upgrades occur).

Generate a key with Keysmith

To generate a key we need to do three steps: generate a seed, create a private/public keypair from that seed, and create an account on the Ledger with that private/public keypair.

Terminal 
keysmith generate -o seed.txt

                                                                
keysmith private-key -o private.pem
Terminal
keysmith account -i 0
dg502c8c2dcd20b9aa5f43fdc2a875bd8f1a183fe289f5c7e3c236b49be1cf44

The command above will generate an unique account on the Ledger (number above is an example outpout).

Connect that key to DFX

DFX is the Canister SDK used for smart contract canister development. This SDK has an important feature we will use: the SDK can send messages to canisters on the Internet Computer. Since we need to send *signed* messages to the Governance Canister on the NNS, we will need to use DFX (neither Quill nor Keysmith can send messages). First install, dfx...

Terminal 
sh -ci "$(curl -fsSL https://sdk.dfinity.org/install.sh)"
dfx --version
dfx 0.8.1

Now that DFX is installed, we need to link the previously created keys by "importing an identity" with a command of structure dfx identity import [flag] $identity_name $pem_file-name.

In the example below, the name of the $identity_name is "proposals" and the name of $pem_file_name is "private.pem"

Output
dfx identity import proposals private.pem
creating identity: proposals...

Install Quill

Quill is a minimalistic ledger and governance toolkit.

We will use Quill to create messages which will then be sent by DFX to the Governance Caniter (Quill cannot send messages to the Internet Computer)

To install Quill...

4.1 Install Rust if you do not have it already: Rust installation

4.2 Install OpenSS if you do not have it already, which we recommend you install via Homebrew

4.3 Follow the Download instructions

.4 Follow the Build instructions (these need Rust)

Once Quill is installed, we make sure the account linked to Quill is the same that was created in Step 1 (in our example, it is dg502c8c2dcd20b9aa5f43fdc2a875bd8f1a183fe289f5c7e3c236b49be1cf44)

Terminal 
target/release/quill --pem-file private.pem public-ids
dfx --identity proposals ledger account-id

Now that Quill and DFX can both sign messagess on behalf of the same account number (example dg502c8c2dcd20b9aa5f43fdc2a875bd8f1a183fe289f5c7e3c236b49be1cf44), then we need to create a neuron and give it at least 1 ICP so that it can submit an NNS Proposal (only Neurons can sjubmit NNS proposals and it costs them 1 ICP)

Create a neuron that will submit the Proposals to the NNS

Buy at least 1.01 ICP and send it to the account number we have been handling. In our example it is: dg502c8c2dcd20b9aa5f43fdc2a875bd8f1a183fe289f5c7e3c236b49be1cf44

Once you have the ICP, you can use the NNS Dapp or any exchange or wallet to send it to the address.

Now that your account has ICP, use quill to spawn a neuron. Remember to write down the neuron ID.

Terminal
target/release/quill --pem-file private.pem neuron-stake --name $NAME --amount $AMOUNT > message.json
target/release/quill send message.json

Increase the neuron's dissolve to at least six months. In our example we will use 1 year which is 31557600 seconds. In our case, the neuron ID is 148885932052941245

Terminal
target/release/quill --pem-file private.pem neuron-manage 148885932052941245 --additional-dissolve-delay-seconds 31557600 > message.json
target/release/quill send message.json

Send Quill messages to NNS via DFX

To create a Motion Proposal we need to have DFX send a message to the Governance Canister on behalf of the newly-created neuron. For the sake of legibility, we show the fields and then the actual (harder to read) dfx command underneath

example fields
{
  "action": "motion",
  "summary": "implement design",
  "url": "https://github.com/ic-association/nns-proposals/blob/main/proposals/governance/20210901T1500Z.md",
  "motionText": "Currently, smart contract canister...",
  "neuron": "{id=1488859320522941245:nat64}",
}

Use DFX to construct a messsage that is sent to the Governance Canister.

Few things to note:

  • "neuronId" has the type "nat64" appended to the number sent via dfx. It will not work otherwise.
  • "motionText" can be be many paragraphs long.
  • "proposals" in the example below is the name of the identity created for this example.
  • "proposals" in the example below is the name of the identity created for this example.
  • "rrkah-fqaaa-aaaaa-aaaaq-cai" is the ID of the Governance Canister: Candid UI
Terminal
dfx --identity proposals canister --network ic --no-wallet call rrkah-fqaaa-aaaaa-aaaaq-cai manage_neuron '(record {id = null; command=opt variant {MakeProposal=record {url="urlExample"; action=opt variant {Motion=record {motion_text="motionExample"}}; summary="summaryExample"}}; neuron_id_or_subaccount=opt variant {NeuronId=record {id=148885932052941245:nat64}}})'

You can now see your proposal in the Internet Computer Dashboard!
